= SQL CLAUSES


SQL clauses help define how data is retrieved, filtered and manipulated. They provide a structured way to specify what data to include, how to organize it and what conditions must be met for rows to be included in the result set.

The following table summarizes the clauses supported by Oxla:

[cols="1,1",options="header"]
|===
| Clause Name  | Description 

|xref:oxla-reference:sql-clauses/from/from.adoc[FROM] |Defines the source table(s) or view(s) for the query
|xref:oxla-reference:sql-clauses/where.adoc[WHERE] |Filters rows based on specified conditions
|xref:oxla-reference:sql-clauses/group-by.adoc[GROUP BY] |Groups rows sharing common values in specified columns for aggregation
|xref:oxla-reference:sql-clauses/having.adoc[HAVING] |Filters grouped rows based on aggregate conditions
|xref:oxla-reference:sql-clauses/order-by.adoc[ORDER BY] |Sorts the result set by specified columns in ascending or descending order
|xref:oxla-reference:sql-clauses/limit.adoc[LIMIT] |Restricts the number of rows returned by the query
|xref:oxla-reference:sql-clauses/offset.adoc[OFFSET] |Skips a specified number of rows before returning results
|xref:oxla-reference:sql-clauses/set-operations/overview.adoc[SET OPERATIONS] |Combine or compare results from multiple `SELECT` statements, such as `UNION`, `INTERSECT` and `EXCEPT`
|xref:oxla-reference:sql-clauses/with.adoc[WITH] |Creates temporary named result sets (Common Table Expressions) for reuse within queries
|xref:oxla-reference:sql-clauses/over-window.adoc[OVER] |Specifies the window over which window functions to operate on subsets of data
|===
