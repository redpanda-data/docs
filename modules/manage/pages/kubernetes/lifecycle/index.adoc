= Lifecycle Management
:description: Manage the lifecycle of Redpanda brokers in Kubernetes, including decommissioning, recovery mode, and rolling restarts.
:page-layout: index
:page-categories: Management
:env-kubernetes: true
:personas: platform_operator, platform_engineer

After your Redpanda cluster is running in Kubernetes, you can manage broker lifecycle operations such as safely removing brokers, recovering from failures, and performing rolling restarts for maintenance.

== Decommission brokers

Decommissioning is the safe and controlled way to remove a broker from the cluster. The process migrates partition replicas to remaining brokers before shutting down the broker, so you avoid data loss or cluster instability.

See xref:manage:kubernetes/k-decommission-brokers.adoc[Decommission Brokers].

== Recovery mode

Recovery mode starts Redpanda with limited functionality so you can repair a failed cluster. In recovery mode, you can inspect and manage partitions while most client-facing operations are disabled.

See xref:manage:kubernetes/tiered-storage/k-recovery-mode.adoc[Recovery Mode].

== Rolling restart

A rolling restart restarts one broker at a time while the remaining brokers continue running. This minimizes downtime during configuration updates, version upgrades, or cluster maintenance.

See xref:manage:kubernetes/k-rolling-restart.adoc[Rolling Restart].
