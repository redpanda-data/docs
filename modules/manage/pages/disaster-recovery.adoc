= Disaster Recovery
:description: Comprehensive guide to implementing disaster recovery strategies for Redpanda clusters.

Disaster recovery ensures your Redpanda deployment continues operating when catastrophic failures occur. This guide covers strategies, implementation patterns, and best practices for multi-region disaster recovery.

== Overview

Redpanda supports multiple disaster recovery patterns:

* **Active/Passive**: One primary cluster with standby replicas
* **Backup and Restore**: Point-in-time recovery using Tiered Storage
* **Cross-region Replication**: Real-time data replication across regions
