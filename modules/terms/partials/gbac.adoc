=== GBAC
:term-name: GBAC
:hover-text: Group-based access control lets you assign permissions to OIDC groups so that users inherit access based on their identity provider group membership.
:category: Redpanda security

GBAC allows you to manage permissions at the group level instead of per user. You can grant permissions to groups in two ways: create ACLs with `Group:<name>` principals, or assign groups as members of RBAC roles. Both approaches can be used independently or together.
