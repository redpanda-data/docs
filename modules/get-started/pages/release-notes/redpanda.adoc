== New commands

* xref:reference:rpk/rpk-profile/rpk-profile-validate.adoc[`rpk profile validate`] - Validate profile configuration and detect common issues such as cloud settings, authentication references, and token expiry.

== New global flag

All `rpk` commands now support the `--ignore-profile` flag, which ignores `rpk.yaml` and `redpanda.yaml` and uses default settings instead.

== Updated commands

The following `rpk` commands have been updated:

* xref:reference:rpk/rpk-cluster/rpk-cluster-quotas-alter.adoc[`rpk cluster quotas alter`] - Now supports three entity types: `user`, `client-id`, and `client-id-prefix`.
* xref:reference:rpk/rpk-cluster/rpk-cluster-storage-restore-start.adoc[`rpk cluster storage restore start`] - Updated to restore full cluster data including metadata. New `--cluster-uuid-override` flag.
* xref:reference:rpk/rpk-profile/rpk-profile.adoc[`rpk profile`] - Documents the `RPK_PROFILE` environment variable for temporarily overriding the current profile.
* xref:reference:rpk/rpk-profile/rpk-profile-prompt.adoc[`rpk profile prompt`] - New `raw` modifier to disable ANSI color escapes.
* xref:reference:rpk/rpk-redpanda/rpk-redpanda-admin-brokers-decommission.adoc[`rpk redpanda admin brokers decommission`] - New `--skip-liveness-check` flag replaces the hidden `--force` flag.
* xref:reference:rpk/rpk-redpanda/rpk-redpanda-admin-config-log-level-set.adoc[`rpk redpanda admin config log-level set`] - New `--help-loggers` flag to display available loggers from the cluster.
* xref:reference:rpk/rpk-registry/rpk-registry-schema-create.adoc[`rpk registry schema create`] - New `--metadata-properties` flag for schema metadata.
* xref:reference:rpk/rpk-registry/rpk-registry-schema-get.adoc[`rpk registry schema get`] - New `--print-metadata` flag for schema metadata properties.
* xref:reference:rpk/rpk-shadow/rpk-shadow-describe.adoc[`rpk shadow describe`] - New `--format` flag for JSON/YAML output and `--print-registry` flag for schema registry configuration.
